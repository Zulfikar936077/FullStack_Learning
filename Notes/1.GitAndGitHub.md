**2. Initialize Repository**  
a. To change the directory:  
cd "C:/....................." (cd means change direcotory)  


*Remember:*      
-Use quation marks ("")    
-Use closing slash (/) if you are not using ("") marks   


b. git init (to create the repo in the local machine)  
c. git status (to check the status of the repo)  
d. git add 1.index.html (to track the untracked file)  
e. git add . (to add all the untracked file)  
f. git rm --cached 1.index.html (to remove any tracked file)    

**3. Commit**  
a. git commit -m "comment"  
b. git add . && git commit -m "comment"   
(If you want to add all untracked files and commit together)


**4. Change files and view differeneces**  
a. git status   
(if a file is modifies in local storage and you want to check it)  
b. git diff  
(to check the diff bet the two versions, tje changed file will be in green)   
c. git add 1.index.html  
 (then if you are satisfied with the change and want to push the file in the staging stage then you can use the command)

  **5. Delete/Remove File**

  **9. Amend commit**

**10. View changes in commits**  
a. git log -p  
(to see the specifics of the commit history)  
**(type q to exit the log)**  

**11. Reset to previous commit**  
git reset c193894  
(c193894 is the number of the previous commit from the log)  

**12. Branches**  
a. git branch (to check the branch)  
b. git branch hasan (to create a new branch, here hasan is the new branch)  
c. git switch hasan (to switch from main branch to hasan branch)  
d. git commit -a -m "Added new file in the new branch"  
(If you open a new file in the new branch and modify it, then only use the command. Here -a is used to skip the add step of the untracked files before commit)  


**13. Merge Branch**  
a. git merge -m "Merge hasan back to the master" hasan  
(Use this merge command if you want to write your own message instead of the default messsage)

**14. Delete Branch**  
a. git branch -d hasan  

**15. Merge Conflict**  
*The following command will switch and create a new branch at the same time:*  
a. git switch -c newBranch  
*Then if you want to merge the newBranch to the main branch then write:  
b. git merge newBranch  
(this merge command is used if you are ok with the default message)  
*After the merge command if you find any conflict*  
-then open the conflict file in notepad  
-keep the version you want  
-save the file  
-use the following command:  
git commit -a -m "Updated index.html"  

**16. Push local repo to GitHub**
(Use the following command when pushing the local repo in GitHub for the first time)  
a. git remote add origin ......................... (https of the repo from the GitHub)  
b. git branch -M main (to set the target branch to main)  
c. git push -u origin main (to push the local repo to the cloud)  
d. git push --all  

**17. Fetch and Pull**  
To get the changes from cloud (GitHub) to the local repo:  
*Option 1*  
a. git fetch (To download all the history from remote tracking branches)  
b. git merge (To merge with what we have on the local repo)  

*Option 2*  
a. git pull (To fetch and merge together)
